Historial

Tus últimas 50 entradas. De un JWT guardamos solo la cabecera —también cuando viaja dentro de un texto más largo: una petición HTTP pegada entera, una cookie, un parámetro de URL—; de un JSON, las claves sin sus valores; de un base64, solo su longitud; de una URL, el dominio. El resto (texto, hashes, UUIDs y timestamps) se guarda tal cual. Reconocemos un JWT porque su cabecera descodifica a un JSON con «alg»: lo que no cumpla eso —un token opaco, algo con forma de JWT que no lo es— se guarda entero.
